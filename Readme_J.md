### 1. イントロダクション：本プロジェクトの「想い」
電気化学インピーダンス分光法（EIS）において、緩和時間分布（DRT: Distribution of Relaxation Times）解析は、 Nyquistプロット上で重なり合った「電荷移動」「拡散」「接触抵抗」などの物理過程を、時定数の違いとして鮮やかに分離し可視化できる極めて強力な手法です。



#### なぜ「Excel VBA」なのか
現在、DRT解析にはMATLABベースの「DRTtools」やPythonのライブラリなど、優れたツールが既に存在します。しかし、多くの製造業や研究機関の現場では、以下のような厳しい**ITセキュリティ制限**という壁が立ちはだかっています。

* **機密データの外部送信禁止**: 開発中の未発表データを、利便性の高いクラウド型の解析アプリにアップロードすることは禁じられています。
* **外部ソフトのインストール禁止**: Python環境の構築や未知のEXEファイルの実行が社内規程で許可されない、あるいは情報システム部門の承認に数ヶ月を要することも珍しくありません。
* **予算の制約**: 高価な有償解析ソフトをチーム全員分導入する予算を確保するのは容易ではありません。

本プロジェクトは、**「追加インストール不要」「完全ローカル処理（オフライン完結）」「中身が透明なVBAソースコード」**に徹底的にこだわりました。現場のエンジニアが、セキュリティポリシーを遵守しながら、今すぐ手元のExcelで高度な解析を完結できることを目指しています。

#### AIとの共創による開発
本プロジェクトのソースコードは、**Googleの生成AI「Gemini」**との対話を通じて構築されました。専門的な物理モデルの実装、数値計算アルゴリズムの最適化、およびクロスプラットフォーム（Mac/Windows）対応のコード生成において、人間とAIの協調作業によって完成したものです。最新のAI技術を「道具」として活用し、伝統的なExcel VBAという環境に高度な解析機能を実装しました。

### 2. 解析パラメータの詳細設定 (Global Settings)
`DRT_core.bas` の冒頭には、解析の精度を左右する独立変数が定義されています。

- **`KK_THRESHOLD`**: データの整合性を評価。装置の不調や外乱による異常値を自動除外します。
- **`LAMBDA_SCAN`**: 正則化パラメータ ($\lambda$) の探索範囲。$10^{0} \sim 10^{-10}$ の広い範囲で最適なバランスを自動探索します。
- **`CUT_LOW_FREQ`**: 低周波端に現れやすい不自然なスパイク（擬似ピーク）を表示からカットします。

---

### 3. 解析コアの内部処理プロセス (Step-by-Step)
マクロ **`ActiveSheetDRT_all`** は、以下の科学的ステップを順次実行します。

1. **KK-Filter**: 測定データが物理的に正しい（線形・安定）かを確認。
2. **Tikhonov正則化**: 数学的に不安定な逆問題を解き、ノイズを抑えつつ真の物理応答を抽出。
3. **L-Curve法**: 「モデルの残差」と「解の滑らかさ」のバランスが理想的な「曲がり角（エルボー点）」を最適 $\lambda$ として自動採用。



4. **モデル再構成**: 算出したDRTからインピーダンスを再計算。実測値と重ねてプロットし、妥当性を検証。

---

### 4. 使用方法 (Usage)

#### A. 解析コア機能（単一シートの解析）
1.  解析したいデータを、シートの **A列:周波数(Hz)、B列:実部(Z')、C列:虚部(Z'')** に準備します。
2.  マクロ **`ActiveSheetDRT_all`** を実行します。
3.  同じシートの A7/G7セル付近に、Nyquist比較図とDRTスペクトル図が生成されます。

#### B. オプション機能（複数ファイルの一括処理）
1.  `Top` シート等の起点となるシートで **`SelectFiles`** を実行し、複数の **`.z` ファイル**（テキスト形式）を選択。
2.  **`InsertTextCsvFiles`** を実行。各ファイルが解析用形式（A-C列）に自動変換・展開。
3.  **`ProcessAllExtSheets`** を実行。全シートの解析を自動で行い、最後に **`Summary_Plots`** シートに集約プロットを生成。

### 5. 動作環境と制約
* **動作確認済み**: macOS 版 Excel。
* **Windows環境**: コードは共通化していますが、実機での詳細な動作確認は未実施です。
* **対応ファイル形式**: 一括インポート機能は **`.z` 拡張子** のテキストファイルのみに対応しています。
